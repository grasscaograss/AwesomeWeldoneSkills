# Martin Fowler · 参考来源

## 核心书籍

- **《Refactoring: Improving the Design of Existing Code》**（1st ed. 1999 with Kent Beck, 2nd ed. 2018）
  - 代码异味分类体系的权威来源
  - 72 种重构手法的完整目录
  - 「任何时候都可以重构」哲学的原始出处

- **《Patterns of Enterprise Application Architecture》**（2002）
  - 企业级架构模式：Active Record、Data Mapper、Repository、Service Layer 等
  - 领域层、数据层分离的经典论述

- **《UML Distilled》**（1st ed. 1997，3rd ed. 2003）
  - 轻量级建模思想

- **《Domain-Specific Languages》**（2010，with Rebecca Parsons）

## martinfowler.com — Bliki 文章精选

Fowler 的「Bliki」（blog + wiki 混合体）是其思想的主要输出平台，关键文章：

- [Code Smell](https://martinfowler.com/bliki/CodeSmell.html) — 代码异味概念的定义
- [Refactoring](https://martinfowler.com/tags/refactoring.html) — 重构主题文章集
- [Continuous Integration](https://martinfowler.com/articles/continuousIntegration.html) — 持续集成实践
- [BeckDesignRules](https://martinfowler.com/bliki/BeckDesignRules.html) — Kent Beck 四条简单设计规则
- [Eradicating Non-Determinism in Tests](https://martinfowler.com/articles/nonDeterminism.html)
- [Feature Branch](https://martinfowler.com/bliki/FeatureBranch.html)
- [Strangler Fig Application](https://martinfowler.com/bliki/StranglerFigApplication.html) — 遗留系统演化模式
- [TechnicalDebt](https://martinfowler.com/bliki/TechnicalDebt.html)
- [Microservices](https://martinfowler.com/articles/microservices.html)（与 James Lewis 合著）

## 著名公开争论与对话

- **「TDD is Dead」系列对话**（2014）
  - 参与者：David Heinemeier Hansson, Kent Beck, Martin Fowler
  - 链接：[Is TDD Dead?](https://martinfowler.com/articles/is-tdd-dead/)
  - 涉及测试哲学、单元测试 vs 集成测试的边界讨论

- **「Workflows of Refactoring」** — OOP 2014 Keynote（YouTube）
- **「Making Architecture Matter」** — OSCON 2015 Keynote
- **ThoughtWorks Technology Radar** — Fowler 作为 Technology Advisory Board 成员的思想体现
  - https://www.thoughtworks.com/radar

## 经典语录来源

- *"Any fool can write code that a computer can understand..."*
  — 《Refactoring》第一版前言，也引用自 Harold Abelson

- *"When you feel the need to write a comment, first try to refactor..."*
  — 《Refactoring》第 1 章 / martinfowler.com

- *"Refactoring is the process of changing a software system in such a way that it does not alter the external behavior of the code yet improves its internal structure."*
  — 《Refactoring》定义

## ThoughtWorks 背景

Martin Fowler 自 2000 年起担任 ThoughtWorks 首席科学家（Chief Scientist），
其思想深刻影响了 ThoughtWorks 的技术实践、咨询方法论与 Technology Radar 内容。

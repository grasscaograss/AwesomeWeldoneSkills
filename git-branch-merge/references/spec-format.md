# opsx:new 需求描述格式

调用 opsx:new 时，给出清晰的需求描述，让 AI 能在目标分支架构上正确实现。

## 需求描述模板

```
## 背景
从 <source> 分支迁移：<spec 标题>
原始提交：<hash1>, <hash2>

## 功能意图
<这个功能要实现什么效果，解决什么问题>

## 参考实现
<源分支的关键实现思路，可以贴出关键代码片段或文件路径>
原文件路径：<source-branch 里对应的文件>

## 目标分支的约束
<目标分支的相关架构特点，需要注意的接口/类/命名约定>

## 验收标准
<实现完成后，如何确认功能正常，通常是 dotnet build 通过>
```

## 示例

```
## 背景
从 develop 分支迁移：修复焊缝模板匹配中的跟踪判断逻辑
原始提交：a881016e

## 功能意图
焊缝模板匹配时，跟踪判断逻辑存在误判，导致某些情况下跟踪状态不正确。
需要修复 IsTracking 的判断条件。

## 参考实现
原文件：src/Weldone.Domain/Welding/ToolPlanning/WeldPoseMapUtils.cs
关键改动：将 IsTracking 的判断从 `> 0` 改为 `>= threshold`

## 目标分支的约束
develop_DualArm 中该文件已有双臂相关修改，需要兼容双臂调用路径，
不能影响现有双臂逻辑。

## 验收标准
dotnet build 通过
```
